# 写作风格要求

**规则**：章节内容必须满足三个要求：图文并茂、理论讲解、内容充分展开。

**Why**：用户反馈当前内容偏总结式、纯文字堆砌，读起来枯燥，与 Notion 中详细展开的讲解风格差距大。

**How to apply**：
1. **图文并茂**：使用 Mermaid 图表（架构图、流程图、对比图），穿插在文字之间，不要集中堆砌
2. **讲解型风格**：需要把理论一步步拆解，使得零基础人员也能读懂
3. **内容展开**：每个概念要展开讲透，不要一句话总结。关键架构要逐层拆解（像 Notion Faster R-CNN 页面那样把 RPN 的每条路径③④⑤都详细解释）
4. 避免纯表格总结、避免"bulleted list"堆砌。用段落式讲解 + 图表辅助
